# 0009. 처방전 원본 이미지는 영구 보관하지 않는다

## Context

처방전 이미지는 다음 정보를 포함한다.
- **고유식별정보**: 환자 주민등록번호 — 개인정보보호법 §24의2에 따라 처리 법령 근거 없으면 수집·저장 금지
- **민감정보(건강정보)**: 처방 약물 — 별도 동의 필수
- **제3자 PII**: 처방의·약사 이름·면허번호

원본을 보관하면 PIPA §24의2 저촉 위험이 상시 존재하고, 유출 사고 시 피해 범위가 크다. 한편 보호자 검증·OCR 재인식·알고리즘 개선 등 "원본을 보관해야 할 명분"으로 흔히 거론되는 시나리오는 모두 마스킹본 또는 사용자 재촬영으로 대체 가능하다.

## Decision

처방전 원본 이미지는 다음 원칙으로 처리한다.

1. **백엔드는 원본을 영구 저장하지 않는다.** OCR·마스킹 처리 후 즉시 폐기한다.
2. **임시 객체로만 NCP에 업로드**한다. NCP Object Storage Lifecycle Policy로 **24시간 만료** 강제. 처리 완료 시 명시 삭제, 실패해도 만료로 자동 정리.
3. **마스킹된 이미지만 영구 보관**한다. 보호자·시니어 열람은 마스킹본 only.
4. **재인식·알고리즘 개선 등 추가 활용은 사용자에게 재촬영 요청**으로 대체. 별도 학습 데이터 활용은 옵트인 동의 + 가명처리 별도 spec.

## Consequences

### 긍정적
- PIPA §24의2(주민번호 처리 금지) 저촉 위험 **0**
- PIPA §3(개인정보보호 원칙) "필요 최소한 수집·보유" 부합
- 데이터 유출 시 피해 범위 최소화 (마스킹본만 노출)
- 사용자 동의서 단순화 ("이미지는 OCR·마스킹 후 즉시 삭제됩니다")
- 사용자 권리(삭제·열람) 행사 단순
- Object Storage 보유 비용 절감

### 부정적
- OCR/AI 알고리즘 개선 시 원본 데이터셋 자체 구축 불가 → 옵트인 별도 동의 흐름 필요
- 사용자가 잘못 인식된 항목을 발견했을 때 백엔드가 원본 비교 디버깅 불가 → 사용자 재촬영 의존
- "원본을 보면 더 잘 알 수 있을 텐데" 같은 보호자 요구는 불가 (마스킹본만 제공)

## Alternatives (considered)

- **(A) 원본 영구 보관 + 강력한 접근통제**
  PIPA §24의2 저촉 위험 상시 존재. 유출 시 책임 무거움. 의료 PII 다루는 스타트업 규모에 과한 보안·법적 리스크.
- **(B) 원본 단기(예: 30일) 보관 + 자동 만료**
  주민번호 등 고유식별정보 보관 자체가 §24의2 위반 여지. 기간이 짧다고 해소되지 않음. 보호자 검증 목적은 마스킹본으로 충분하다는 분석에 따라 기각.
- **(C) 클라이언트(앱) 디바이스에만 원본 보관, 백엔드에 안 보냄**
  최선의 선택이지만 OCR/마스킹을 백엔드가 수행하려면 어쨌든 원본이 한 번은 백엔드 메모리·임시 스토리지에 도달해야 함. 본 결정의 24시간 만료 임시 업로드 + 즉시 폐기가 (C)의 정신을 충족하는 실용 절충안.
